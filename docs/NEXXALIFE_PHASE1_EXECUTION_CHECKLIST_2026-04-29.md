# NexxaLife — checklist de execução fase 1

Data: 2026-04-29

---

## Objetivo da Fase 1

Subir a fundação inicial do novo produto sem carregar a dívida estrutural do legado.

---

## Checklist

### Descoberta
- [ ] Ler a auditoria completa
- [ ] Listar todas as telas do legado
- [ ] Listar todos os domínios do produto
- [ ] Identificar dependências de storage antigo

### Arquitetura
- [ ] Definir estrutura-alvo de diretórios
- [ ] Definir namespaces do módulo
- [ ] Definir convenção old/new
- [ ] Definir quais telas entram na primeira onda

### UX/UI
- [ ] Mapear componentes herdáveis
- [ ] Mapear componentes a recriar
- [ ] Definir shell visual do novo módulo
- [ ] Definir padrões de página

### Dados e domínio
- [ ] Listar entidades principais
- [ ] Definir contratos iniciais
- [ ] Marcar o que é mock temporário
- [ ] Marcar o que exige engine real

### Saída mínima da fase
- [ ] docs estratégicos criados
- [ ] estrutura base definida
- [ ] primeira onda priorizada

---

## Não fazer nesta fase

- Não portar o storage antigo como solução final
- Não recriar todas as telas de uma vez
- Não implementar engine completa do diagnóstico sem arquitetura
- Não misturar backlog de produto com improviso de UI
